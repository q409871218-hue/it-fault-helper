> 格式：问题分类 | 关键词 | 原因 | 解决步骤 | 注意事项

## 🔌 硬件故障

### 电脑无法开机
- **关键词**：无法开机、不启动、按下电源没反应
- **可能原因**：
  1. 电源线未插好或电源故障
  2. 主板电池没电
  3. 内存条接触不良
  4. 硬盘/显卡等硬件故障
- **解决步骤**：
  1. 检查电源线是否插紧，插座是否有电
  2. 拔掉所有外设，只留电源线测试
  3. 打开机箱，拔插内存条（用橡皮擦金手指）
  4. 尝试清除 CMOS（拔掉主板电池，短接清除针脚）
  5. 如仍不行，怀疑电源或主板故障，需送修
- **注意事项**：操作前先断电；静电可能损坏硬件

### 显示器无信号
- **关键词**：显示器黑屏、无信号、检测信号线
- **可能原因**：
  1. 视频连接线（HDMI/VGA/DP）松动或损坏
  2. 视频输出选择错误（集成显卡/独立显卡）
  3. 显卡故障或未正确安装
  4. 显示器故障
- **解决步骤**：
  1. 检查视频线两端是否插紧，换一根线测试
  2. 尝试切换视频输出源（键盘 Win+P 选择）
  3. 重新插拔显卡
  4. 如有集成显卡，尝试接集成显卡输出口
  5. 找另一台显示器测试
- **注意事项**：确认是显示器问题还是主机问题

### 蓝屏错误
- **关键词**：蓝屏、BSOD、BugCheck
- **常见错误代码**：
  - `0x0000007B`：硬盘模式不兼容（进BIOS改SATA模式）
  - `0x000000D1`：驱动问题（最近装了什么驱动？）
  - `CRITICAL_PROCESS_DIED`：系统文件损坏
  - `MEMORY_MANAGEMENT`：内存故障
- **解决步骤**：
  1. 记录蓝屏错误代码
  2. 搜索该错误代码的解决方案
  3. 最近是否安装了新硬件/软件/更新？
  4. 尝试进入安全模式卸载最近安装的程序
  5. 运行 `sfc /scannow` 修复系统文件
- **注意事项**：保存蓝屏截图有助于诊断

## 🌐 网络故障

### 无法上网（有线）
- **关键词**：无法上网、网络受限、本地连接受限、感叹号
- **可能原因**：
  1. 网线未插好或网线故障
  2. DHCP获取IP失败
  3. DNS服务器问题
  4. 网卡驱动异常
  5. 路由器/交换机故障
- **解决步骤**：
  1. 检查网线两端是否插紧
  2. 禁用再启用网卡
  3. 运行 `ipconfig /release` → `ipconfig /renew` 重新获取IP
  4. 修改DNS为 114.114.114.114 或 8.8.8.8
  5. 检查路由器/光猫指示灯是否正常
  6. 重启路由器/光猫
- **注意事项**：记录具体错误信息（IP是否获取到？）

### WiFi无法连接
- **关键词**：WiFi连不上、搜索不到、已保存但连不上
- **可能原因**：
  1. 无线开关/飞行模式未关闭
  2. 密码错误
  3. 信号太弱
  4. 无线网卡驱动问题
  5. 路由器连接数已满
- **解决步骤**：
  1. 检查笔记本无线开关或按 Fn+航空键
  2. 忘记网络重新输入密码连接
  3. 靠近路由器测试
  4. 更新无线网卡驱动
  5. 重启路由器
  6. 检查设备管理器中无线网卡是否正常
- **注意事项**：有些企业WiFi需要证书认证

### 网络受限（能上网但速度慢）
- **关键词**：网络受限、网速慢、卡
- **可能原因**：
  1. DNS解析慢
  2. 网线质量差（百兆网线接千兆口）
  3. 路由器带机量过大
  4. 有人蹭网
- **解决步骤**：
  1. 修改DNS为快速DNS（114.114.114.114）
  2. 检查网线是否为超五类/六类线
  3. 重启路由器
  4. 登录路由器检查连接设备
- **注意事项**：定期更换WiFi密码

## ☁️ 云平台/服务器端口故障

### 云终端无法连接服务器
- **关键词**：云终端连不上、无法登录、无响应
- **可能原因**：
  1. 网络不通（检查网线、交换机）
  2. 服务器端口被阻断
  3. 服务器服务未启动
  4. 防火墙阻止
- **解决步骤**：
  1. 检查云终端网络配置（IP、网关、DNS）
  2. 用 ping 测试服务器是否可达
  3. 检查服务器关键端口是否开放（见下方端口表）
  4. 在服务器上检查对应服务状态
  5. 检查防火墙规则
- **注意事项**：云平台常见端口冲突或未开放

### 客户端无法下载镜像
- **关键词**：镜像下载失败、7011端口、网络传输
- **可能原因**：
  1. 镜像分发服务（7011端口）未启动
  2. 网络带宽不足
  3. 磁盘广播冲突
- **解决步骤**：
  1. 检查 7011 端口是否可访问
  2. 检查 IO Service 服务状态
  3. 检查磁盘广播设置（7008端口）
  4. 确认客户端 7707 端口正常通信
- **注意事项**：多线程下载可能占用大量带宽

### VNC远程桌面无法连接
- **关键词**：VNC连不上、远程桌面、7791端口
- **可能原因**：
  1. VNC服务未启动
  2. 7791端口被阻断
  3. 防火墙拦截
  4. 5900 VNC标准端口被占用
- **解决步骤**：
  1. 检查服务器 7791（WEB VNC）和 5900（标准VNC）端口
  2. 确认 VNC Server 服务运行中
  3. 检查防火墙规则是否允许 7791/5900 端口
  4. 尝试重启 VNC 服务
- **注意事项**：云平台通常使用 7791 而非标准 5900

### DHCP获取IP失败
- **关键词**：DHCP、无法获取IP、67/68端口
- **可能原因**：
  1. DHCP服务未启动（67端口）
  2. 网线连接问题
  3. DHCP地址池耗尽
  4. 恶意DHCP服务器干扰
- **解决步骤**：
  1. 检查服务器 67 端口（DHCP服务）和 68 端口（客户端）状态
  2. 确认 DHCP 服务运行中
  3. 检查地址池是否还有可用IP
  4. 使用 Wireshark 抓包分析 DHCP 过程
- **注意事项**：PXE网络启动依赖DHCP，需同时检查4011端口

### 客户端心跳异常
- **关键词**：心跳、掉线、7700端口
- **可能原因**：
  1. 网络不稳定
  2. 7700 心跳端口被阻断
  3. 服务器端 IO Service 异常
  4. 客户端 UI 监听端口（7706）异常
- **解决步骤**：
  1. 检查 7700（心跳）和 7706（UI监听）端口通信
  2. 确认服务器端服务正常运行
  3. 检查网络延迟和丢包
  4. 查看客户端日志定位断连原因
- **注意事项**：持续掉线可能表明网络拥塞或服务器过载

## 💻 IDV云桌面故障（联想）

### IDV客户端无法启动/黑屏
- **关键词**：IDV黑屏、不启动、云桌面无显示
- **可能原因**：
  1. 镜像下发未完成
  2. 网络不通（客户端无法联系服务器）
  3. 服务器端服务异常
  4. BIOS中启动顺序错误
- **解决步骤**：
  1. 检查客户端是否走完PXE启动流程
  2. 确认客户端与服务器网络互通（ping服务器IP）
  3. 登录管理端检查该客户端状态
  4. 检查服务器7000/7001端口服务是否运行
  5. 重启客户端或重新下发镜像
- **注意事项**：IDV与TCI架构不同，IDV走PXE启动流程

### IDV镜像下发失败
- **关键词**：镜像下载失败、下发超时、镜像同步
- **可能原因**：
  1. 7011端口镜像分发服务未启动
  2. 镜像文件过大，网络超时
  3. 客户端磁盘空间不足
  4. 7008磁盘广播端口不通
- **解决步骤**：
  1. 确认服务器7011端口可访问
  2. 检查IO Service服务状态
  3. 确认客户端磁盘容量足够
  4. 尝试使用U盘离线写入方式部署镜像
  5. 检查网络交换机是否支持巨帧 Jumbo Frame
- **注意事项**：建议使用千兆网络环境部署镜像

### IDV授权错误/注册失败
- **关键词**：授权错误、License、注册、7007-7020
- **可能原因**：根据错误代码判断（见下方的授权错误代码表）
- **解决步骤**：
  1. 对照授权错误代码表判断具体错误类型
  2. 联系供应商获取正确的License文件
  3. 确认服务器硬件序列号与License匹配
  4. 检查License是否在有效期内
- **注意事项**： License与服务器硬件绑定，更换硬件需要重新申请

### IDV客户端无法远程唤醒
- **关键词**：唤醒、开机、WOL、跨VLAN
- **可能原因**：
  1. 客户端BIOS中WOL功能未开启
  2. 跨VLAN唤醒时交换机未配置UDP广播转发
  3. 网卡电源管理中"允许计算机关闭此设备"被勾选
  4. 客户端断电状态下无法唤醒
- **解决步骤**：
  1. 进入客户端BIOS，开启Wake on LAN / Remote Wake Up
  2. 确认客户端关机后保持通电（网卡的指示灯应亮）
  3. 跨VLAN时在交换机上配置UDP-helper
  4. 在Windows网卡属性中关闭"关机网络唤醒"的节能选项
- **注意事项**：WOL需要客户端保持待机供电，断电无法唤醒

### IDV系统更新后异常
- **关键词**：更新后蓝屏、更新失败、回退
- **可能原因**：
  1. 镜像更新包与当前系统不兼容
  2. 更新过程中断
  3. 驱动冲突
- **解决步骤**：
  1. 登录管理端，使用"生成新桌面快照"回退
  2. 检查更新日志定位错误
  3. 确认是否使用了覆盖升级（会清除用户数据）
  4. 建议先在测试机上验证更新包
- **注意事项**：覆盖升级会清除用户数据，标准升级保留数据

### IDV客户端磁盘空间不足
- **关键词**：磁盘满、C盘满、空间不足、VHD
- **可能原因**：
  1. 系统镜像模板分区太小
  2. 更新补丁累积占用空间
  3. 临时文件未清理
- **解决步骤**：
  1. 使用VHD镜像分区扩容工具扩展C盘
  2. 清理Windows更新缓存（cleanmgr /dism）
  3. 重新制作更大容量的镜像模板
  4. 使用"添加数据磁盘"功能扩展存储
- **注意事项**：扩容前建议备份重要数据

### IDV客户端蓝屏或死机
- **关键词**：IDV蓝屏、死机、卡logo
- **可能原因**：
  1. 镜像模板驱动不兼容
  2. 硬件配置与模板不一致（不同型号机器用同一镜像）
  3. 镜像文件损坏
- **解决步骤**：
  1. 记录蓝屏错误代码并搜索解决方案
  2. 确认客户端硬件配置是否与模板制作时一致
  3. 重新制作镜像模板，使用通用驱动
  4. 检查服务器端镜像文件完整性（SHA校验）
- **注意事项**：不同CPU/显卡配置的机器需要不同的镜像模板

### IDV网络克隆掉线
- **关键词**：克隆、批量部署、网络装系统、PXE
- **可能原因**：
  1. DHCP过VLAN配置不正确
  2. TFTP服务未启动（69端口）
  3. 镜像文件过大导致传输超时
- **解决步骤**：
  1. 检查核心交换机是否配置了UDP-helper
  2. 确认DHCP中继指向正确的服务器IP
  3. 在交换机上配置ip helper-address
  4. 检查69/4011端口是否放通
- **注意事项**：网络克隆需要交换机支持UDP广播转发

## 🔧 跨VLAN网络唤醒配置

### 交换机配置要点

**Cisco交换机：**
```
ip directed-broadcast
```

**H3C/HUAWEI/3COM交换机：**
```
# 全局
sys
ip forward-broadcast
udp-helper enable
udp-helper port 7

# 进入客户机所在VLAN接口
interface vlan 100
ip forward-broadcast
```

### 客户端WOL硬件设置

**BIOS中需要开启的选项：**
- Wake on LAN / Power on LAN
- PME Event WakeUp
- Resume by MAC LAN
- Wake-Up by PCI Card
- 由PCI/PCI-E设备唤醒

**不同品牌主板位置：**
- Dell：Power Management Setup → Remote Wake Up → On
- 联想：Power → Automatic Power ON → Wake On LAN → Enabled
- 华硕：高级 → 高级电源管理 → 开启由PCI/PCI-E设备唤醒
- 技嘉：BIOS功能 → 网络启动 → Legacy First
- 微星：STANDARD mode → Setup Wake Up Configuration → Resume By PCI-E Device

**Windows网卡设置：**
- 关机网络唤醒：开启
- 魔术封包唤醒：开启
- 样式比对唤醒：开启
- 环保节能/节能以太网络：**关闭**
- "允许计算机关闭此设备以节约电源"：**关闭**

## 💻 软件故障

### 程序无响应/卡死
- **关键词**：程序卡死、无响应、停止运行
- **可能原因**：
  1. 程序本身bug
  2. 内存不足
  3. 磁盘空间不足
  4. 与其他程序冲突
- **解决步骤**：
  1. 等待30秒看是否自动恢复
  2. 按 Ctrl+Shift+Esc 打开任务管理器结束进程
  3. 检查内存和磁盘空间使用情况
  4. 关闭不必要的后台程序后重试
  5. 重新安装程序
- **注意事项**：记得保存未保存的工作

### 系统运行缓慢
- **关键词**：电脑卡、反应慢、系统慢
- **可能原因**：
  1. 磁盘空间不足（<10%）
  2. 启动项过多
  3. 病毒/恶意软件
  4. 内存不足
  5. 系统更新中
- **解决步骤**：
  1. 打开任务管理器查看CPU/内存占用
  2. 磁盘清理（cleanmgr）
  3. 禁用不必要的启动项（msconfig → 启动）
  4. 全盘杀毒
  5. 考虑增加内存或换固态硬盘
- **注意事项**：机械硬盘变慢可能是坏道前兆

### 软件报错/弹窗
- **关键词**：报错、弹窗、应用程序错误
- **解决步骤**：
  1. 记录完整错误信息
  2. 搜索该错误信息
  3. 尝试以管理员身份运行
  4. 兼容模式运行（右键 → 属性 → 兼容性）
  5. 重新安装软件
- **注意事项**：错误代码是搜索的关键

## 🖨️ 外设故障

### 打印机无法打印
- **关键词**：打印机不能打印、打印队列卡住
- **可能原因**：
  1. 打印机脱机
  2. 打印队列卡住
  3. 驱动问题
  4. 连线松动
- **解决步骤**：
  1. 检查打印机是否开机、是否脱机
  2. 打开"设备和打印机"，删除卡住的队列
  3. 在打印机属性中"使用打印机脱机"取消勾选
  4. 删除打印机重新添加驱动
  5. 重启打印服务（services.msc → Print Spooler）
- **注意事项**：共享打印机要注意权限设置

### 客户机D盘（重新格式化）
客户机PXE网络启动进入BootLoader界面后，在管理端：
1. 勾选目标客户机（可批量）
2. 远程命令 → 命令名称随意
3. 命令内容：`dd if=/dev/zero of=/dev/nvme0n1 bs=2GB count=1`
   - `nvme0n1` 为实际磁盘名（BootLoader界面可查看，如显示sda则用 `/dev/sda`）
4. 点"确定"执行，命令执行后需重启客户机生效
