# 常见IT故障知识库

> 本文件由管理员维护，遇到已知故障时优先匹配，未命中再搜索网络。
> 格式：问题分类 | 关键词 | 原因 | 解决步骤 | 注意事项

---

## 🔌 硬件故障

### 电脑无法开机
- **关键词**：无法开机、不启动、按下电源没反应
- **可能原因**：
  1. 电源线未插好或电源故障
  2. 主板电池没电
  3. 内存条接触不良
  4. 硬盘/显卡等硬件故障
- **解决步骤**：
  1. 检查电源线是否插紧，插座是否有电
  2. 拔掉所有外设，只留电源线测试
  3. 打开机箱，拔插内存条（用橡皮擦金手指）
  4. 尝试清除 CMOS（拔掉主板电池，短接清除针脚）
  5. 如仍不行，怀疑电源或主板故障，需送修
- **注意事项**：操作前先断电；静电可能损坏硬件

### 显示器无信号
- **关键词**：显示器黑屏、无信号、检测信号线
- **可能原因**：
  1. 视频连接线（HDMI/VGA/DP）松动或损坏
  2. 视频输出选择错误（集成显卡/独立显卡）
  3. 显卡故障或未正确安装
  4. 显示器故障
- **解决步骤**：
  1. 检查视频线两端是否插紧，换一根线测试
  2. 尝试切换视频输出源（键盘 Win+P 选择）
  3. 重新插拔显卡
  4. 如有集成显卡，尝试接集成显卡输出口
  5. 找另一台显示器测试
- **注意事项**：确认是显示器问题还是主机问题

### 蓝屏错误
- **关键词**：蓝屏、BSOD、BugCheck
- **常见错误代码**：
  - `0x0000007B`：硬盘模式不兼容（进BIOS改SATA模式）
  - `0x000000D1`：驱动问题（最近装了什么驱动？）
  - `CRITICAL_PROCESS_DIED`：系统文件损坏
  - `MEMORY_MANAGEMENT`：内存故障
- **解决步骤**：
  1. 记录蓝屏错误代码
  2. 搜索该错误代码的解决方案
  3. 最近是否安装了新硬件/软件/更新？
  4. 尝试进入安全模式卸载最近安装的程序
  5. 运行 `sfc /scannow` 修复系统文件
- **注意事项**：保存蓝屏截图有助于诊断

## 🌐 网络故障

### 无法上网（有线）
- **关键词**：无法上网、网络受限、本地连接受限、感叹号
- **可能原因**：
  1. 网线未插好或网线故障
  2. DHCP获取IP失败
  3. DNS服务器问题
  4. 网卡驱动异常
  5. 路由器/交换机故障
- **解决步骤**：
  1. 检查网线两端是否插紧
  2. 禁用再启用网卡
  3. 运行 `ipconfig /release` → `ipconfig /renew` 重新获取IP
  4. 修改DNS为 114.114.114.114 或 8.8.8.8
  5. 检查路由器/光猫指示灯是否正常
  6. 重启路由器/光猫
- **注意事项**：记录具体错误信息（IP是否获取到？）

### WiFi无法连接
- **关键词**：WiFi连不上、搜索不到、已保存但连不上
- **可能原因**：
  1. 无线开关/飞行模式未关闭
  2. 密码错误
  3. 信号太弱
  4. 无线网卡驱动问题
  5. 路由器连接数已满
- **解决步骤**：
  1. 检查笔记本无线开关或按 Fn+航空键
  2. 忘记网络重新输入密码连接
  3. 靠近路由器测试
  4. 更新无线网卡驱动
  5. 重启路由器
  6. 检查设备管理器中无线网卡是否正常
- **注意事项**：有些企业WiFi需要证书认证

### 网络受限（能上网但速度慢）
- **关键词**：网络受限、网速慢、卡
- **可能原因**：
  1. DNS解析慢
  2. 网线质量差（百兆网线接千兆口）
  3. 路由器带机量过大
  4. 有人蹭网
- **解决步骤**：
  1. 修改DNS为快速DNS（114.114.114.114）
  2. 检查网线是否为超五类/六类线
  3. 重启路由器
  4. 登录路由器检查连接设备
- **注意事项**：定期更换WiFi密码

## 💻 软件故障

### 程序无响应/卡死
- **关键词**：程序卡死、无响应、停止运行
- **可能原因**：
  1. 程序本身bug
  2. 内存不足
  3. 磁盘空间不足
  4. 与其他程序冲突
- **解决步骤**：
  1. 等待30秒看是否自动恢复
  2. 按 Ctrl+Shift+Esc 打开任务管理器结束进程
  3. 检查内存和磁盘空间使用情况
  4. 关闭不必要的后台程序后重试
  5. 重新安装程序
- **注意事项**：记得保存未保存的工作

### 系统运行缓慢
- **关键词**：电脑卡、反应慢、系统慢
- **可能原因**：
  1. 磁盘空间不足（<10%）
  2. 启动项过多
  3. 病毒/恶意软件
  4. 内存不足
  5. 系统更新中
- **解决步骤**：
  1. 打开任务管理器查看CPU/内存占用
  2. 磁盘清理（cleanmgr）
  3. 禁用不必要的启动项（msconfig → 启动）
  4. 全盘杀毒
  5. 考虑增加内存或换固态硬盘
- **注意事项**：机械硬盘变慢可能是坏道前兆

### 软件报错/弹窗
- **关键词**：报错、弹窗、应用程序错误
- **解决步骤**：
  1. 记录完整错误信息
  2. 搜索该错误信息
  3. 尝试以管理员身份运行
  4. 兼容模式运行（右键 → 属性 → 兼容性）
  5. 重新安装软件
- **注意事项**：错误代码是搜索的关键

## 🖨️ 外设故障

### 打印机无法打印
- **关键词**：打印机不能打印、打印队列卡住
- **可能原因**：
  1. 打印机脱机
  2. 打印队列卡住
  3. 驱动问题
  4. 连线松动
- **解决步骤**：
  1. 检查打印机是否开机、是否脱机
  2. 打开"设备和打印机"，删除卡住的队列
  3. 在打印机属性中"使用打印机脱机"取消勾选
  4. 删除打印机重新添加驱动
  5. 重启打印服务（services.msc → Print Spooler）
- **注意事项**：共享打印机要注意权限设置

## ⚠️ 维护提示

### 定期维护建议
- 每月：清理磁盘、更新系统补丁
- 每季度：全面杀毒、检查硬件健康
- 每年：清灰、更换导热硅脂

### 常用诊断命令
```bash
# 网络诊断
ipconfig /all           # 查看IP配置
ipconfig /release       # 释放IP
ipconfig /renew         # 重新获取IP
ipconfig /flushdns      # 刷新DNS缓存
ping 8.8.8.8            # 测试网络连通性
tracert www.baidu.com   # 路由追踪

# 系统信息
msinfo32               # 查看系统信息
devmgmt.msc            # 设备管理器
services.msc            # 服务管理
diskmgmt.msc           # 磁盘管理
```
